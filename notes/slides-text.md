- Based on the input and output modalities, we can classify MLLMs into categories like:
- Audio-text-to-text
- Image-text-to-text
- Image-text-to-image
- Image-text-to-video
- Video-text-to-text
- Any-to-any
- Hugging Face is also using these.
- MLLM emphasizes the alignment of latent spaces between different modalities. We are extremely familiar with the classic encoder-decoder networks like UNet, thus, it is very intuitive that we can emerge multiple encoders and decoders in different combinations to perform different types of tasks. ===== SLIDE 5 =====
- Introduction to MLLM
- Align vision features to an existing LLM
- Compress vision through learned queries
- Inject vision through cross-attention
- Train a unified multimodal autoaggressive model
- Recall the training process of a naive UNet. It is essentially training a pair of mappings:
- Encoder: Input -&gt; Latent
- Decoder: Latent -&gt; Output
- We know the optimization process is inherently stochastic, which makes the latent space nondeterministic across trials. Therefore, two independently trained encoder and decoder do not share the same latent space. To align them, there are several approaches listed on the right.

===== SLIDE 16 =====
- MedGemma 1.5
- 3D CT Scan
- 2D Slices
- Text Outputs
- MedSigLIP
- Encoder
- Gemma 3
- Decoder
- MedGemma 1.5 4B is based on Gemma 3 with the same general architecture, using a 400M MedSigLIP vision encoder as the visual front end and a decoder-only Transformer LLM as the text generator. Images are normalized to 896×896 and encoded into 256 visual tokens per image.
- 
- The key improvement between MedGemma 1 and MedGemma 1.5 is the long context window, with which we are able to feed multiple uniformly sampled slices (up to 85, modeled as a time sequence) to represent a 3D volume. However, in this mini course, we only deal with 2D images.
- 
- Note that it processes each 896×896 image as 4 448×448 tiles and each tile yields 64 visual tokens.
- Cited: Sellergren, A., Gao, C., Mahvar, F., Kohlberger, T., Jamil, F., Traverse, M., ... &amp; Golden, D. (2026). Medgemma 1.5 technical report. arXiv preprint arXiv:2604.05081. https://doi.org/10.48550/arXiv.2604.05081

- Common ways to fine-tune large models include:
- Full fine-tuning
- Feature extraction

To adapt the target domain, we need to apply fine-tuning. There are multiple ways to perform fine-tuning. ===== SLIDE 23 =====
- Fine-tuning Protocols
- Full fine-tuning
- Update all model parameters on a task-specific dataset
- Feature extraction
- Freeze the foundation model and train only a task-specific output head
- Adapter tuning
- Insert small trainable adapter modules between existing layers
- Prefix tuning
- Learn task-specific prefix vectors that guide the model’s hidden states
- Prompt tuning
- Train soft prompt embeddings while keeping the base model frozen

===== SLIDE 36 =====
- Key Takeaways
- MLLMs combine multiple modalities, such as images, text, video, and audio, with this course focusing mainly on medical image-text-to-text models.
- The core challenge in MLLMs is modality alignment, because vision encoders and language models do not naturally share the same latent representation space.
- Different MLLM architectures solve alignment differently, including LLaVA-style projection, BLIP-2 learned queries, Flamingo-style cross-attention, and Kosmos-style unified multimodal modeling.
- MLLMs have several clinical use cases, including perception, reasoning, documentation, triage, and patient-facing support.
- Clinical tasks covered include report generation, longitudinal comparison, multi-class classification, multi-label classification, and regression.
- The course uses MedGemma 1.5 4B as the main example, a LLaVA-style medical vision-language model designed for image-text-to-text tasks.
- MedGemma 1.5 combines a MedSigLIP vision encoder with a Gemma 3 decoder-only language model, turning medical images into visual tokens that the LLM can use to generate text.
- The hands-on workflow uses the FLARE-MLLM-2D dataset, which is downloaded, preprocessed, and converted into Hugging Face supervised fine-tuning format.
- Fine-tuning is done with QLoRA, a parameter-efficient method that keeps the base model mostly frozen while training small low-rank adapter weights on compressed 4-bit model weights.
- Model performance is evaluated using CRIMSON and GREEN scores, which assess radiology report quality by checking for clinical errors, missing findings, false findings, and severity-weighted mistakes.
